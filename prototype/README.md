# 不秃 · Web 交互原型

这是根据《计划软件 Web MVP PRD v0.1》制作的本地交互原型，用于验证信息架构和关键交互，不是正式产品代码。

## 已覆盖的体验

- 五个一级页面：今日、目标、计划、复盘、设置
- 任意页面快速记录，事项进入收集箱
- 可展开/收起、增加、删除的目标树，目标节点会显示已分配的日期或周期
- 统一的计划项编辑器：任务、收集箱事项和目标都可修改归属、角色、状态与排期
- 月视图整月网格、周视图「本周计划 + 本周重点 + 两列七天」、日视图分时段计划
- 只排到月/周的计划独立展示，可继续拖进某一天；日期任务也可在日期格之间拖动改期
- 未定时间任务放在今日侧栏，并可拖入或拖出小时日历
- 拖拽排期，以及不依赖拖拽的按钮入口
- 待重新安排、拆分和放弃
- 日复盘摘要与引导问题
- 由用户自己填写、点击即可修改的「本周一句话」
- 本地备份和设置入口演示

## 本地打开

在项目根目录启动一个静态文件服务：

```powershell
python -m http.server 4173 --directory .\prototype
```

然后在浏览器访问：

```text
http://127.0.0.1:4173
```

原型不需要安装 npm 包，也不会连接服务器或发送数据。

## 运行测试

```powershell
node --test .\prototype\model.test.mjs
```

测试覆盖快速记录、状态切换、重新排期历史、父计划进度、逾期判断和目标路径。

## 原型限制

- 页面刷新后恢复演示数据，不做正式持久化。
- 导入、导出和清空数据只演示入口，不执行真实文件操作。
- 时间块拉伸采用点击增加 15 分钟的简化交互。
- 视觉用于确认方向，后续仍需经过用户反馈和正式设计系统整理。
