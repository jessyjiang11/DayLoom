# DayLoom v0.4 Online MVP Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** 在保留 v0.3 原型的同时，建立 React + TypeScript + Supabase 的在线 Web MVP，提供邮箱登录、个人数据隔离、云端保存、实时同步和编辑型计划本 UI。

**Architecture:** `web/` 是 Vite React SPA，通过 Supabase JS 使用 Auth、PostgreSQL 和 Realtime；`supabase/migrations/` 保存可审计的数据库与 RLS；TanStack Query 管理服务器状态和缓存。GitHub Actions 构建 `web/dist` 并部署到 GitHub Pages。

**Tech Stack:** React, TypeScript, Vite, React Router, TanStack Query, Supabase JS, Zod, dnd-kit, date-fns, Vitest, Testing Library, Playwright, PostgreSQL/RLS。

### Task 1：建立 React + TypeScript 工程

**Files:**
- Create: `web/`
- Create: `web/.env.example`
- Modify: `.gitignore`

**Steps:**
1. 运行 `pnpm create vite@latest web --template react-ts`。
2. 安装运行依赖：`pnpm --dir web add @supabase/supabase-js @tanstack/react-query react-router-dom zod date-fns @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities`。
3. 安装测试依赖：`pnpm --dir web add -D vitest jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event playwright`。
4. 设置 `vite.config.ts` 的 GitHub Pages base 为 `/DayLoom/`，并配置 Vitest `jsdom`。
5. 新建 `.env.example`，只写 `VITE_SUPABASE_URL` 与 `VITE_SUPABASE_PUBLISHABLE_KEY` 名称，不写真实值。
6. 在 `.gitignore` 排除 `web/.env*`，但保留 `web/.env.example`。
7. 运行 `pnpm --dir web build`，预期构建成功。
8. 提交：`git commit -m "build: scaffold DayLoom web app"`。
