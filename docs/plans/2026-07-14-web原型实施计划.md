# Web Prototype Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Goal:** Build a local, clickable desktop-first Web prototype that validates quick capture, goal hierarchy, drag scheduling, humane rescheduling, and lightweight review.

**Architecture:** Use a dependency-free static web application in `prototype/`. Keep domain transitions in a pure ES module so they can be tested with Node's built-in test runner. Keep the interface state in memory and seed it with realistic sample data; this is a UX prototype, not the production persistence layer.

**Tech Stack:** HTML5, CSS custom properties, vanilla JavaScript ES modules, Node.js built-in test runner.

---

### Task 1: Create the prototype shell and design system

**Files:**
- Create: `prototype/index.html`
- Create: `prototype/styles.css`

**Steps:**
1. Define the five-item navigation: 今日、目标、计划、复盘、设置.
2. Add the persistent quick-capture control and main content mount point.
3. Define color, typography, spacing, radius, shadow, focus, empty, and error tokens.
4. Implement desktop layout and a compact responsive fallback.
5. Verify the HTML loads without external assets and key controls have accessible labels.

### Task 2: Implement and test the domain model

**Files:**
- Create: `prototype/model.mjs`
- Create: `prototype/model.test.mjs`

**Steps:**
1. Write tests for quick capture, status changes, rescheduling history, and automatic parent progress.
2. Run `node --test prototype/model.test.mjs` and verify tests fail before implementation.
3. Implement pure helpers for task lookup, creation, status transition, scheduling, and progress.
4. Run the tests and verify they pass.

### Task 3: Implement the five prototype screens

**Files:**
- Create: `prototype/app.mjs`
- Modify: `prototype/index.html`
- Modify: `prototype/styles.css`

**Steps:**
1. Seed realistic plan data based on the planning-app project itself.
2. Render the 今日 screen with goal path, focus tasks, unscheduled strip, hour calendar, inbox, and rescheduling queue.
3. Render the 目标 screen with an expandable hierarchy and detail panel.
4. Render the 计划 screen with month/week/day tabs and scheduling buckets.
5. Render the 复盘 and 设置 screens with useful prototype states.
6. Wire navigation, quick capture, state changes, selection, and screen rerendering.

### Task 4: Add interaction prototypes

**Files:**
- Modify: `prototype/app.mjs`
- Modify: `prototype/styles.css`

**Steps:**
1. Add HTML drag-and-drop from inbox/unscheduled items into day and hour targets.
2. Add reschedule actions: 今天、本周、拆分、放弃.
3. Add time-block resizing affordance as a visual prototype and click-based duration control.
4. Add undo toast after scheduling changes.
5. Add keyboard-visible focus and non-drag menu alternatives.

### Task 5: Verify behavior and visuals

**Files:**
- Create: `prototype/README.md`

**Steps:**
1. Run all model tests and verify success.
2. Start a local static server using the bundled Python runtime.
3. Open the prototype in a browser automation session and capture the 今日、目标、计划 screens.
4. Check overflow, contrast, focus, empty states, and drag targets at desktop and narrow widths.
5. Fix visible defects, rerun tests, and document how to launch the prototype.

### Acceptance criteria

- The prototype launches locally without installing packages.
- All five primary screens are reachable.
- Quick capture adds a visible inbox item.
- A plan item can be scheduled without changing its parent goal.
- An overdue item can be moved to today or abandoned.
- Parent progress updates when a child is completed.
- The interface remains usable without drag-and-drop through button alternatives.
- Node tests pass and visual inspection shows no blocking layout defects.
